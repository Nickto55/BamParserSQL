# BamParserSQL

Программа для автоматизированного создания сводной таблицы на основе данных из SQL-сервера и Excel-файла. Предназначена для обработки и обогащения данных по ДСЕ (Деталь и Сборочная Единица) в контексте задач БАМ.

## Описание

Приложение считывает список ДСЕ из предоставленного Excel-файла (лист "Изделия"), выполняет пакетные запросы к базе данных SQL Server для получения отсутствующей технологической и производственной информации, а затем обновляет исходный Excel-файл новыми данными. 

Инструмент оснащен графическим интерфейсом (GUI), системой логирования в реальном времени, расчетом оставшегося времени выполнения и защитой от зависаний при длительных операциях.

## Основные возможности

- **Чтение и запись Excel**: Автоматическая обработка файлов `.xlsx` и `.xls` (лист "Изделия").
- **Интеграция с SQL Server**: Пакетная отправка запросов к базе данных для оптимизации времени обработки.
- **Обогащение данных**: Автоматическое добавление колонок:
  - *ТП не в архиве*
  - *ДСЕ без маршрутов*
  - *ДСЕ без основного материала*
  - *ДСЕ без трудоемкости*
  - *Всего нет УП*
  - *Наименование изделия (ИС)*
- **Умное логирование**: Отображение прогресса, расчетного и оставшегося времени выполнения.
- **Контроль длительных операций**: Предупреждение и возможность отмены, если расчетное время превышает 30 минут.
- **Безопасная остановка**: Возможность прервать выполнение с сохранением промежуточных результатов.

## Технологический стек

- **Язык**: Python 3.8+
- **Работа с данными**: `pandas`, `numpy`, `openpyxl`, `xlrd`
- **База данных**: `pyodbc` (или аналогичный драйвер для SQL Server)
- **GUI**: `tkinter`, `CustomTkinter`, `CTkMessagebox`
- **Конфигурация**: `python-dotenv`

## Установка и настройка

### 1. Клонирование репозитория
```bash
git clone https://github.com/Nickto55/BamParserSQL.git
cd BamParserSQL
```

### 2. Создание виртуального окружения (рекомендуется)
```bash
python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для macOS/Linux:
source venv/bin/activate
```

### 3. Установка зависимостей
```bash
pip install pandas numpy openpyxl xlrd python-dotenv customtkinter CTkMessagebox pyodbc
```
*(Примечание: для работы `pyodbc` на некоторых ОС может потребоваться предварительная установка драйверов ODBC для SQL Server)*

### 4. Настройка переменных окружения
Создайте в корневой папке проекта файл `.env` и заполните его данными для подключения к вашей базе данных:
```env
SQL_SERVER=ваш_сервер
SQL_DB=ваша_база_данных
SQL_EXC=дополнительный_параметр_подключения
```

## Использование

1. Подготовьте входной Excel-файл. Убедитесь, что в нем есть лист с именем **"Изделия"**, содержащий столбец **"Дсе"** с перечнем обрабатываемых единиц.
2. Запустите основной скрипт:
   ```bash
   python bam_parcer_sql.py
   ```
3. В открывшемся графическом интерфейсе выберите путь к вашему Excel-файлу.
4. При необходимости настройте параметры пакетной обработки (разделение запросов).
5. Нажмите кнопку запуска и следите за прогрессом в окне логирования.
6. После завершения работы исходный Excel-файл будет автоматически обновлен новыми данными.

## Конфигурация

Помимо `.env`, приложение использует внутренние конфигурационные файлы (управляемые через `handlings/handling_config.py`) для сохранения настроек интерфейса, путей к последним файлам и параметров обработки запросов.
